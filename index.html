Library Documentation

Your guide to using the Library effectively

Note: This documentation provides detailed descriptions and return values for each function in the Library.
Important: Ensure that all required parameters are passed correctly to avoid unexpected errors.
Tip: Use meaningful names for your titles and variables to keep your code understandable and maintainable.
Warning: Improper handling of callback functions can lead to unexpected behavior. Always validate input parameters.

Library.New(title)

Title: <string> - Title of the UI
Return: Library <table> 
local myLibrary = Library.New("My Awesome UI")

Library:Toggle()

Return: nil - Toggles UI 
myLibrary:Toggle()

Library:Notify(Title, Text, Position, Direction, Callback)

Title: <string> - Title of notification
Text: <string> - Description of notification
Position: <UDim2> - Position of start of the notification
Direction: <UDim2> - Direction of the notification
Callback: <function>
Return: (value)
Value: <boolean> - Accepted or cancelled 
myLibrary:Notify("Notice", "This is a notification", UDim2.new(0,0,0,0), UDim2.new(1,0,1,0), function(value)
    print("Notification result:", value)
end)

Library:AddPage(Title, Icon)

Title: <string> - Title of page
Icon: <string> - Asset ID of icon (optional)
Return: Pages <table> 
local myPage = myLibrary:AddPage("Home", "rbxassetid://123456")

Pages:AddSection(Title)

Title: <string> - Title of section
Return: Sections <table> 
local mySection = myPage:AddSection("Main Section")

Pages:AddSearchBar()

Return: nil - Adds Search Bar to the page 
myPage:AddSearchBar()

Sections:AddButton(Title, Callback)

Title: <string> - Title of button
Callback: <function>
Return: button <Instance> 
mySection:AddButton("Click Me", function()
    print("Button clicked")
end)

Sections:AddToggle(Title, IsEnabled, Callback)

Title: <string> - Title of toggle
IsEnabled: <boolean> - Default value of toggle
Callback: <function>
Return: (value, update)
Value: <boolean> - New toggle value
Return: toggle <Instance> 
mySection:AddToggle("Enable Feature", true, function(value)
    print("Toggle value:", value)
end)

Sections:AddTextBox(Title, Callback)

Title: <string> - Title of textbox
Callback: <function>
Return: (text, focus)
Text: <string> - Textbox content
Focus: <boolean> - Focus lost signal
Return: textbox <Instance> 
mySection:AddTextBox("Enter Text", function(text)
    print("Text entered:", text)
end)

Sections:AddKeybind(Title, Key, Callback)

Title: <string> - Title of keybind
Key: <keycode> - Default binded key
Callback: <function>
Return: (update)
Key: <KeyCode> - Key pressed
Return: keybind <Instance> 
mySection:AddKeybind("Bind Key", Enum.KeyCode.F, function()
    print("Key pressed")
end)

Sections:AddSlider(Title, Value, Min, Max, FixValues, Callback)

Title: <string> - Title of slider
Value: <number> - Default value of slider
Min: <number> - Minimum value of dragger
Max: <number> - Maximum value of dragger
FixValues: <boolean> - If textbox values are bigger than max or smaller than min, it will set to default max or min value
Callback: <function>
Return: (value, update)
Value: <number> - Value of slider
Return: slider <Instance> 
mySection:AddSlider("Volume", 50, 0, 100, true, function(value)
    print("Slider value:", value)
end)

Sections:AddDropdown(Title, Entries, Callback)

Title: <string> - Title of dropdown
Entries: <table> - Content of dropdown
Callback: <function>
Return: (value)
Value: <string> - Text of button clicked
Return: dropdown <Instance> 
local dropdownEntries = {"Option 1", "Option 2", "Option 3"}
mySection:AddDropdown("Select Option", dropdownEntries, function(value)
    print("Selected value:", value)
end)

Additional Notes:

  • Ensure that your callbacks are well-defined to handle all possible return values.
  • Use descriptive titles for your UI elements to maintain clarity and ease of use.
  • Refer to the official documentation for more detailed examples and use cases.

Important Considerations:

When using Library:Notify, be mindful of the position and direction to avoid overlapping notifications. Properly managing the UI state with Library:Toggle can enhance user experience significantly.
